# Evidence Folder

Store sanitized CWQ launch-hardening evidence references here when evidence does not already belong in a repo-specific evidence directory.

Do not store:

- secrets
- raw provider payloads
- images containing personal data
- prompts/responses with user data
- production credentials
- unsanitized logs

Prefer linking to repo-owned evidence paths when they already exist, especially under `fitaly-backend/evidence/`.
